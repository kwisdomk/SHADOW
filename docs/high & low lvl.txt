Core System DefinitionShadow is a silent, autonomous, on-device + cloud-assisted AI security intelligence system built to protect Kenyan users from fraud, scam, spam, phishing, and digital social engineering in real time.Unlike traditional scam filters, Shadow functions as:A. Passive InterceptorContinuously monitors high-risk communication channels without disrupting normal user behavior:SMS / MMS WhatsApp notifications Telegram Email notifications USSD prompts Browser links Call metadata (future scope) Screenshot/image uploads Voice snippets (future) B. Adaptive Threat Intelligence EngineInstead of only blocking known scams:Detects new patterns Learns from user decisions Builds localized threat memory Understands Kenyan Swahili + Sheng + code-switching Adjusts confidence based on region-specific fraud behavior 2. Device-Level Technical Flow (Android First)Layer 1: Data Ingestion Layer (Silent Capture)Android Components:• Notification Listener ServiceReads incoming messages from apps silently.• SMS Retriever / SMS Broadcast ReceiverIntercepts direct SMS.• Accessibility Service (carefully scoped)Optional advanced phishing/USSD detection.• Local Metadata ParserExtracts:Sender Timestamp App source Message body URL Attachments 3. Preprocessing Layer (Edge AI)Before cloud:Functions:• Language DetectionEnglish Swahili Sheng Code-mixed • Token NormalizationExamples:“umewin 50k” → “ume win 50k” “tuma fare” → fraud intent patterns • Threat Pattern ParsingDetect:Urgency Impersonation Reward bait Financial requests PIN/OTP extraction attempts 4. Local Lightweight AI Engine (Fast Path)Goal:Instant under-200ms classification.Model:Small quantized transformer / distilled classifier.Output:SAFE SPAM SUSPICIOUS HIGH RISK Why:Most decisions happen locally to preserve:Speed Privacy Battery 5. AMD Cloud Escalation Layer (Heavy Path)If uncertain:Trigger threshold:Confidence < defined threshold OR novel pattern.Escalate to AMD MI300X backend.AMD-Specific WorkloadsA.
